yoru.css は、 ダーク・テック方向に振り切った日本製の 1 ファイル CSS パッケージです。

// 01 — philosophy

なぜ yoru.css なのか

01
01 / PHILOSOPHY

ダークに、振り切る。

ライトテーマはサポートしません。夜にしか輝かないデザインに全力を注ぐことで、フレームワーク化できない個性を生みます。

02 / SIMPLICITY

1 ファイル、ビルド不要。

<link> 1 行で完結。Webpack も PostCSS も Tailwind も不要。HTML とテキストエディタさえあれば動きます。

03 / FOR YOU

俺だけの CSS。

汎用フレームワークではありません。LP・コーポレート・ポートフォリオの 3 用途に特化した、あなたひとりの道具です。

note: yoru.css は Tailwind や Bootstrap のような網羅性を目指しません。 「この方向で作るなら、これだけあれば足りる」という実用の輪郭を持ったパッケージです。

STEP 01

ファイルを置く

yoru.css を HTML と同じフォルダに置くだけ。CDN も npm も不要です。 

project/
├─ index.html
└─ yoru.css
STEP 02

1 行追加する

<head> にこれを書いたら、もう始まっています。 

<link rel="stylesheet" href="yoru.css">
STEP 03

クラスを使う

クラス名を当てるだけで、ダーク・テックの世界が立ち上がります。

最小のテンプレート

<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>My Site</title>
  <link rel="stylesheet" href="yoru.css">

<body class="has-navbar">

  <section class="hero">
    <div class="container">
      <div class="hero-content">
        <span class="tagline">// hello</span>

        <p>ここから始めよう。</p>
        <a href="#" class="btn btn-primary btn-lg">スタート</a>
      </div>
    </div>
  </section>

</body>
</html>

用途とトーンに合わせて 6 種。サイズは 3 段階。

<button class="btn btn-primary">Primary</button>
<button class="btn btn-outline-cyan btn-lg">大きめ</button>
<button class="btn btn-icon btn-secondary">⚙</button>

カード

標準 / アクセント / ホバーグロー / 画像付き / 横並び。

アクセント

上辺にラインが入ります。

Hover Lift

ホバーで浮きます。

<div class="card card-accent card-hover-lift">
  <h5>タイトル</h5>
  <p>本文</p>
</div>

フォーム

入力要素はすべてデフォルトでスタイル済み。

送信後、確認メールが届きます。

  </span>

ナビバー & モバイルドロワー

固定ナビ + ハンバーガー。モバイルで自動切り替え。

<nav class="navbar">
  <div class="container">
    <a class="navbar-logo">Logo</a>
    <div class="navbar-nav">
      <a href="#">About</a>
      <a href="#">Works</a>
    </div>
    <button class="hamburger" id="hamburger">
      <span></span><span></span><span></span>
    </button>
  </div>
</nav>

<div class="nav-drawer" id="navDrawer"> ... </div>
JS 最小実装： hamburger.addEventListener('click', () => { hamburger.classList.toggle('is-open'); navDrawer.classList.toggle('is-open'); }) の 1 行相当で OK。demo.html に完全版があります。

モーダル

data-modal-open="ID" で開き、data-modal-close で閉じる。

<button data-modal-open="myModal" class="btn btn-primary">開く</button>

<div class="modal-overlay" id="myModal" data-modal-close>
  <div class="modal" onclick="event.stopPropagation()">
    <div class="modal-header">
      <h3 class="modal-title">タイトル</h3>
      <button class="modal-close" data-modal-close>×</button>
    </div>
    <div class="modal-body">...</div>
    <div class="modal-footer">
      <button class="btn btn-primary" data-modal-close>OK</button>
    </div>
  </div>
</div>

アコーディオン

<details> を使うので JS 不要。FAQ に最適。

JS は本当にいらないの？
いりません。ネイティブの <details> の挙動を活かしているので、読み込み順や SEO の問題もなし。
複数同時に開ける？
はい。グループ化したい場合は name 属性を使ってください（ブラウザ対応状況に注意）。
<div class="accordion">
  <details class="accordion-item" open>
    <summary>質問</summary>
    <div>回答</div>
  </details>
</div>

ツールチップ

data-tip="..." 属性だけで CSS のみ実装。

<button data-tip="説明">上（デフォルト）</button>
<button data-tip="説明" class="tip-bottom">下</button>

トースト / 通知

右下に積まれる通知。自動で消えます。

<!-- 挿入先 -->
<div class="toast-stack" id="toastStack"></div>

<!-- JS -->
yoruToast({ type: 'success', title: '完了', body: '保存しました' });

ローディング / スケルトン

スピナー 3 色 × 3 サイズ。スケルトンは shimmer 付き。

<span class="spinner spinner-cyan"></span>

<div class="skeleton skeleton-title"></div>
<div class="skeleton skeleton-text"></div>

スクロール演出

.reveal を付けて IntersectionObserver で .is-visible を付与。

<div class="reveal">上からふわっと</div>
<div class="reveal reveal-left">左からスライド</div>

<script>
const io = new IntersectionObserver(entries => {
  entries.forEach(e => e.isIntersecting && e.target.classList.add('is-visible'));
});
document.querySelectorAll('.reveal').forEach(el => io.observe(el));
</script>

カスタマイズ例

}

よくある質問

07
明るいテーマに対応させたいのですが…
yoru.css は意図的にダーク専用です。色の階調はすべて「夜を前提」に組まれています。 ただし、<body class="theme-soft"> を付けると、やや明るめのダーク（たとえば日中のオフィスでも見やすいトーン）に切り替わります。 完全な白背景が必要な場合は、別の CSS を併用してください。
Tailwind と併用できますか？
技術的には可能ですが、思想が真逆なのでおすすめしません。 yoru.css は「クラスを足していくアトミック」ではなく「コンポーネント名を書くセマンティック」の設計です。 両方を混ぜると、どちらの強みも失います。
JavaScript はどこまで必要ですか？
モーダル・ドロワー・トースト・スクロールリビールの 4 つだけ、わずかな JS が必要です。 コード量は合計 50 行程度 で、demo.html の末尾にそのまま使える完成版が入っています。コピーして </body> の直前に貼るだけで全部動きます。
本番環境で使っても大丈夫ですか？
はい。外部依存は Google Fonts の読み込みのみで、それ以外は完全にセルフコンテインドです。 フォントをローカルホストに変えれば、外部通信ゼロでも動作します。
カスタマイズの指針は？
変数でやる、クラスは増やさない。これが一番の原則です。 :root を上書きすれば色・スペーシング・シャドウまで一貫して変わります。 個別のクラスに style を書くより、変数を直せば済むケースがほとんどです。
ファイルサイズは？
非圧縮で約 50KB、gzip すると 10KB 前後です。Google Fonts の読み込みが別途走りますが、Tailwind 等と比べれば桁違いに軽量です。
